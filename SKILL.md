# NotebookLM Prep — синтези + чисті джерела

Перетворює купу deep-research експортів на готовий до вставлення пакет для
NotebookLM/Gemini. **Дані не переписуються** — тільки прибираються URL,
citation-маркери й хвостові розділи «Джерела». Кожен синтез виходить окремим
копі-текстом; усі посилання зводяться в один унікальний список під ліміт.

## Коли запускати
- Користувач дав ≥2 дослідницьких файли (`.md`/`.txt`) або вставив тексти й хоче
  підготувати їх для NotebookLM / Gemini.
- Згадки: «нотбуклм», «notebooklm», «звести синтези», «очистити джерела»,
  «унікальні посилання», «копі-кнопки», «diepresearch», «ліміт 300 джерел».

## Що робить движок (`scripts/build.py`)
Для кожного джерела:
- **Чистий синтез** → `<PREFIX>_<Label>_copy_ready.txt` (URL, `[^n]`,
  markdown-лінки, PUA-citation-токени ChatGPT і хвостовий розділ
  «Джерела/Sources/References» прибрано; **формулювання збережено дослівно**).
- **Посилання** → один `<PREFIX>_external_unique_<N>_URLs.txt`: усі URL, дедуп за
  нормалізованим host+path, у порядку пріоритету, tracking-параметри зрізано.
- **Виключені домени** → `<PREFIX>_removed_excluded_domains_<N>.txt` (лише якщо
  користувач задав домени для виключення).
- **Звіт** → `<PREFIX>_source_report.txt` (скільки посилань з кожного файлу,
  унікальних, математика ліміту, топ-домени).
- **HTML copy center** → `<PREFIX>_copy_center.html`: вкладки по синтезах +
  вкладка «Посилання», у кожній кнопка «Копіювати». Офлайн, light/dark.

Розпізнає 4 типові формати цитат автоматично: Parallel (інлайн `[… — URL]`),
Perplexity (нумеровані футноти внизу), Gemini (розділ «Джерела»), ChatGPT
(службові PUA-токени `citeturn…` без URL).

## Робочий процес

### 1. Зібрати вхідні дані
- Файли на диску → бери шляхи як є.
- Користувач ВСТАВИВ текст у чат → запиши кожен синтез у окремий файл
  (напр. у папку виводу або scratchpad) і використовуй його шлях.

### 2. ⚠️ СПОЧАТКУ УТОЧНИТИ (обов'язково, до генерації)
Постав користувачу питання через `AskUserQuestion`:

1. **Групування за темами.** «Це одна тема чи кілька? Якщо кілька (напр. окремо
   *тренди*, окремо *цінова дисципліна*) — які файли до якої теми?» Генеруй
   **окремий пакет на кожну тему** (окремий `out_dir` і `PREFIX`). Не змішуй теми
   в один пул, якщо користувач цього явно не просив.
2. **Пріоритет джерел.** «Який пріоритет посилань при дедуплікації?» (перше в
   списку виграє при дублях). Дефолт — у порядку, як користувач дав файли, або від
   найбагатшого на посилання.
3. **Домени для виключення.** «Чи є сайти, які прибрати з посилань (напр. власні
   домени)?» — приймай список, нічого не зашивай у код. Якщо ні — лишай порожнім.
4. **Ліміт.** Дефолт 300 (NotebookLM). Питай, лише якщо є сумнів.

### 3. Побудувати config.json
Створи конфіг (схема — `references/config.md`) з абсолютними шляхами.
Для КОЖНОГО синтезу, де користувач вставив «повний» текст із виносками-цифрами,
приклеєними до слів (напр. `розкоші11`), додай `"strip_inline_digits": true`.
Для звичайних `.md`-експортів — не став (щоб не чіпати `Y2K`, `SS27`, роки).

### 4. Запустити движок
```bash
python3 "<SKILL_DIR>/scripts/build.py" /abs/path/config.json
```
Скрипт друкує JSON-підсумок (кількості, унікальні, запас) — на stdout.

### 5. Показати результат
- Виведи **таблицю**: по кожному файлу скільки посилань відібрано, скільки
  унікальних усього, `N синтезів + M URL = разом / ліміт / запас`.
- Дай клікабельні шляхи до `_copy_center.html` і txt-файлів.
- Якщо `trimmed > 0` — чесно скажи, скільки найнижчих за пріоритетом посилань
  відсіклося під ліміт.

## Важливі нюанси
- **Нічого не вигадувати й не скорочувати.** Тільки видалення технічних елементів.
- **ChatGPT `.md`-експорт часто без URL** (лише `citeturn…`-токени). Тоді синтез
  почиститься, але посилань з нього буде 0 — попроси користувача вставити «повний»
  варіант із блоком посилань текстом, якщо треба врахувати і його джерела.
- **Ліміт 300.** Якщо `синтези + URL > ліміт`, движок відсікає найнижчі за
  пріоритетом URL і пише про це у звіт.
- **Виключення доменів — тільки з відповіді користувача.** Ніколи не хардкодь.
